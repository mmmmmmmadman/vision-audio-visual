# CV Seq 進化方向 - 快速摘要

## 核心發現

### 當前系統評估
- 實時性強 (30 FPS 同步，48kHz 音訊)
- 視覺呈現偏簡陋 (主要是線條和圓圈)
- 互動性不足 (無拖拉編輯，無預設管理)
- 資訊密度低 (無完整序列圖，無統計資訊)

### 最高優先改進 (按建議順序)

#### 1. 序列步進網格 ★★★ (1-2天)
- 顯示完整的 SEQ 序列步進 [●○○○●○○○]
- 高視覺衝擊，低開發成本
- 直接解決「看不到完整序列」的痛點
- 位置: Column 1 下方

#### 2. 波形示波器增強 ★★★ (2-3天)
- 在示波器上疊加 SEQ 步進點和 ENV 曲線
- 提升調試體驗
- 修改 scope_widget.py

#### 3. CV 參數編輯網格 ★★ (3-4天)
- 表格式快速編輯所有參數
- 支援預設管理
- 完整的工作流體驗

#### 4. 統計和分析面板 ★ (1-2天)
- 實時 CV 統計 (Min/Max/Avg/Variance)
- 異常檢測
- 診斷工具

### 中期目標 (1-2月)
- 2D Lissajous 可視化 (SEQ1/SEQ2 相位關係)
- 完整的圖形化 CV 編輯環境
- 多軌編排視圖 (類 DAW)

### 長期願景 (3-6月)
- AI 輔助 CV 設計 (模式識別、生成)
- 與專業 DAW 對標的編排能力
- 實時表演的精細控制

## 決策點 (討論時需確認)

1. **視覺風格**: 保持極簡黑色？或引入漸變/新色系?
2. **互動模式**: 只支援點擊？或支援拖拉？
3. **使用場景**: 實時表演/工作室編輯/教學？
4. **性能目標**: 維持 30 FPS？支援 GPU 加速？
5. **色彩編碼**: 現有粉色/白色/紅色保留？
6. **預設系統**: JSON/YAML/Binary 格式？
7. **序列步數**: 固定 8 步？還是動態 1-32？
